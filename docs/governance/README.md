# FassadenFix Governance-Dokumentation

**Version:** 1.0
**Datum:** 2026-01-13
**Autor:** Manus AI

## Übersicht

Dieses Verzeichnis enthält die vollständige Governance-Dokumentation für das FassadenFix SharePoint Intranet. Alle Dokumente sind direkte Ableitungen der **Manus-optimierten Anweisungsvorlage (Version 1.6)** und der übergeordneten **Master Instruction (Stufe 1)**.

Die Dokumentation stellt sicher, dass das SharePoint-Intranet als **alleinige Single Source of Truth** für Organisations- und Prozesswissen etabliert und dauerhaft gepflegt wird – für Menschen und Maschinen gleichermaßen.

## Dokumentenübersicht

| Dokument | Beschreibung | Zielgruppe |
| :--- | :--- | :--- |
| [SharePoint-Setup-Anweisung](sharepoint_setup_anweisung.md) | Technische Anleitung zur Einrichtung und Erweiterung des SharePoint-Intranets. | IT-Administratoren, SharePoint-Administratoren |
| [NotebookLM-Ingest-Anweisung](notebooklm_ingest_anweisung.md) | Prozess zur Aufnahme von Intranet-Inhalten in Google NotebookLM. | Wissensmanager, KI-Administratoren |
| [Voice-Agent-Prompt-Policy](voice_agent_prompt_policy.md) | Verbindliche System-Prompts und Verhaltensregeln für Sprachassistenten. | KI-Entwickler, Voice-Agent-Administratoren |
| [Migrations- und Pflege-Governance](migrations_und_pflege_governance.md) | Regeln für die Migration von Altdaten und die fortlaufende Pflege des Intranets. | Alle Mitarbeiter, Inhaltsverantwortliche |

## Hierarchie der Dokumente

Die Governance-Dokumente folgen einer klaren Hierarchie:

```
Master Instruction (Stufe 1)
└── Manus 1.6 System Instruction (Stufe 2)
    ├── SharePoint-Setup-Anweisung
    ├── NotebookLM-Ingest-Anweisung
    ├── Voice-Agent-Prompt-Policy
    └── Migrations- und Pflege-Governance
```

Bei Widersprüchen gilt immer das übergeordnete Dokument.
